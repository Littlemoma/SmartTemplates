SmartTemplates :: Screenshots

  • 2 box solution in reply and forward cases (Version 0.9.2pre20 or later)

    ATTENTION: In Version 0.9.2pre20 we have changed the design of SmartTemplates into a 2 box solution (for reply and forward cases ONLY) to improve support for Thunderbird's signature settings.

    TIP: None HTML Example Content for "Quote Header" Box:
    %from(name)% wrote at %date%:

    Instead of an none HTML Quote Header you can configure HTML too

    TIP: None HTML Example Content for "Template" Box:
    Hello %from(name)%,
    Thanks for your message
    regards,
    %ownname%
    %sig%

    Instead of a plain text template you can configure HTML too

    ATTENTION: Please make sure to put your Template in the Template box and your Header in the Quote Header box!
    The option "Use instead of default quote header" should be checked if you have your own Quote Header defined (otherwise you get two headers)!
    If you use %sig% or %sig(2)%, please put it into the Template section.


  • Recommended Settings in Composition&Addressing Window (Version 0.9.1 or later)

    ATTENTION: In Version 0.9.1 we are taking the settings for Composition&Addressing as for Thunderbird.
    If you have an unusual placement of your Template, please check your settings as in the follow screenshot.
    Especially "start my reply above the quote" and place my signature "below the quote (recommended)"
    Please see the current Bugs for this:

    • [Bug 25093]: Signatur missing when replying below quote
    • [Bug 25002]: SmartTemplates Signature file display 2 times??

  • Conversion Window (Version 0.9.1 or later)

    ATTENTION: In Version 0.9.1 we have changed the design for the curled bracket expression and changed the internal Namespace.
    If you install 0.9.1 or later at the first time, you get the conversion Window and in this you should click on "OK Button",
    this converts your old Template to the new Design and copied to the new Namespace, otherwise you can't use your old Templates again!

  • Debug Settings

    TIP: Right-clicking the 'Debug Mode' checkbox opens the advanced debug settings - these are by default deactivated.
    When you are reporting problems we might require you to activate certain ones via double click.
    This will add more detailed output to the javascript error console, helping us to help you better.

  • Help Widget

    TIP: Clicking the "Help Button" opens a help window - here you can select variables by clicking them.
    SmartTemplates will conveniently copy them into the currently edited template.
    Keyboard Focus is also moved back to the template so you can continue typing.
    Creating a template couldn't be easier!
